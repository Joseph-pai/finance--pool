# 湖泊注入成員收入池 (Lake to Pond A) 實作計畫

此功能將允許「系統管理員」與「湖泊管理員」在「湖泊管理頁面」中，自由調撥湖泊資金並注入給任何家庭成員的「收入池（Pond A）」，支援「即時到帳」與「預約到帳」。

## User Review Required

> [!IMPORTANT]
> 關於「預約到帳」，目前系統的結算邏輯主要是以「實際確認 (Confirmed)」為主。為了保持架構最單純並讓所有數值無縫對齊，以下是建議的資料庫寫入策略，請確認是否符合您的期望：
> 
> **提案：**
> 1. **即時到帳**：在前端同時發送兩筆資料：一筆 `status = 'confirmed'` 的 `income_items`（讓成員立刻看到錢入帳並更新收入池），加上一筆 `transactions` 交易（立即從湖泊扣除這筆款項）。
> 2. **預約到帳**：在前端發送一筆 `status = 'pending'` 的 `income_items`（帶有預設日期），並在其欄位標註這是來自湖泊。此時 **湖泊的實際現金暫不會立刻扣除**（因為還沒真正發出去），但成員會在「我的池塘」看到有一筆等待入帳的預期收入。當日期到了、成員按下「確認到帳」時，前端才會去觸發湖泊扣除！
> 
> 如果您希望「預約到帳」時**也要立刻鎖住/扣除湖泊的餘額**，我們則需要較大幅度的資料庫欄位修改（例如引入預先扣款機制）。使用上述的提案將能重用目前的資料庫架構並保持最高穩定度！請問您偏好上述提案嗎？

## Proposed Changes

### 1. Database Schema Updates (Migration 009)

為了讓後續追蹤方便，我們將為 `income_items` 加上一個標籤，表示這筆收入來源是否來自湖泊。

#### [NEW] `009_add_lake_source_to_income.sql`
- 新增 `source` 欄位到 `income_items` 表，預設為 `external` (外部收入)。對於這項新功能產生的紀錄，將標記為 `lake`。
- 新增觸發器：如果這筆 `income_item` 的來源是 `lake` 且變成了 `confirmed`，就自動在 `transactions` 補上一筆扣除湖泊的動作。這也是最萬無一失的做法，確保將來使用者「確認入帳」時湖泊餘額 100% 同步扣款！

### 2. Frontend Updates

我們將在湖泊管理頁面上方提供新的注入選單，並在頁面載入時取得所有的 Family Members。

#### [MODIFY] `src/app/lake/page.tsx`
- **權限與資料提取**：判斷如果是 Admin 或 Lake Manager，就在上方區塊提供「調撥資金至成員」的操作面板。抓取所有的 `profiles`（成員清單）。
- **新增 UI 表單**：
  - 下拉選單：選擇收款成員。
  - 數字輸入框：調撥金額。
  - 選項切換框：「即時入帳」或「預約日期」。
- **API 邏輯**：
  - 直接透過 supabase-js 推送資料進 `income_items`（並夾帶特殊的 `source: 'lake'` 欄位），剩下的扣款邏輯交由資料庫在後端幫我們算到好！

## Verification Plan

### Manual Verification
- 以管理員身份登入，操作「湖泊管理」，選擇一名成員並設定調撥 `$1,000` 為「即時到帳」。
- 重新整理後，確認湖泊餘額被扣了 `$1,000`，且該成員的 `Pond A` (收入池) 的當前餘額確實增加了 `$1,000`。
- 再測試「預約到帳」，確認湖泊餘額暫不扣除，但該成員那裡出現了待確認的入帳款項。然後到「收入管理」將其切為已到帳，再回來看湖泊是否正確扣除了 `$1,000`。
